if pod gets restart, ip can be changed...

service -> give stable ip to pods and dns name... x3 service types available..

1|cluster ip :- used for internal communication of the cluster | cant access via outside of cluster
2|node port :- to access applictaions outside via cluster, this has port range, ip + port
3|load balancer :- production level external access, (public ip assign + cloud providers)

nginx deployment create...
>kubectl apply -f service_deployment.yml

expose it as a service...
>kubectl expose deployment nginx-deploy --port=80 --type=ClusterIP
>kubectl get services

create test pod...
>kubectl run test-pod --image=busybox --restart=Never -- sleep 3600
>kubectl exec -it test-pod -- sh

via test pod, access the service...
>wget -qO- http://nginx-deploy
